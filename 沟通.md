完全正确，而且你已经开始理解整个流程了。

你现在做的是 **ASR（Automatic Speech Recognition，自动语音识别）**。

实际上很多视频网站（B站、YouTube、腾讯课堂、慕课等）的字幕都是这样生成的。

整个流程就是：

```
        视频(mp4)
            │
            ▼
      提取音频（Audio）
            │
            ▼
   faster-whisper 语音识别
            │
            ▼
      带时间轴的文字
      (segments)
            │
      ┌─────┴──────┐
      ▼            ▼
    TXT         SRT字幕
                    │
                    ▼
          给视频添加字幕
```

------

# 你可以理解成有三种输出

## 第一种：TXT（纯文本）

例如：

```
大家好，今天我们学习SpringBoot。

首先创建项目。

然后配置数据库。
```

适合：

- AI总结
- RAG知识库
- 学习笔记

------

## 第二种：SRT（字幕）

例如：

```
1
00:00:00,000 --> 00:00:04,200
大家好，今天我们学习SpringBoot。

2
00:00:04,200 --> 00:00:07,800
首先创建项目。
```

播放器可以直接加载。

例如：

```
VLC
PotPlayer
MPC-HC
```

都支持：

```
视频.mp4
视频.srt
```

自动显示字幕。

------

## 第三种：直接烧录到视频（Hard Subtitle）

也就是：

```
MP4
      +
SRT
      │
      ▼
新MP4（字幕已经嵌进去）
```

例如：

原视频

```
老师讲课
```

↓

处理以后

```
老师讲课
—————————————
今天我们学习SpringBoot
```

字幕已经在画面里面了。

一般就是用 **FFmpeg** 完成。

例如：

```
ffmpeg -i test.mp4 \
-i test.srt \
-c:v libx264 \
-c:a copy \
-c:s mov_text \
output.mp4
```

以后我们完全可以自动完成。

------

# 其实 Whisper 输出的不是一句一句

而是：

```
segments
```

每一个 segment 都包含：

```
segment.start

segment.end

segment.text
```

例如：

```
{
    start: 0.0,
    end: 3.52,
    text: "大家好"
}
```

所以：

TXT、

SRT、

ASS、

VTT、

都是根据这些数据生成。

## 方案一：外挂字幕，最推荐

这种方式不会重新压制视频，速度最快，也不损失画质。

先把：

```
outputs\test.srt
```

复制到视频旁边，并改成与视频相同的名字：

```
D:\faster_whisper\
├── test.mp4
└── test.srt
```

然后使用 PotPlayer、VLC 等播放器打开 `test.mp4`，一般会自动加载字幕。

如果没有自动加载，可以在播放器中手动选择：

```
字幕 → 添加/选择字幕 → test.srt
```

这种字幕可以随时关闭、修改，适合自己看课程视频。

------

## 方案二：把字幕永久嵌入视频

这种叫“硬字幕”或“烧录字幕”。字幕会成为视频画面的一部分，发给别人时也能直接看到。

### 第一步：安装 FFmpeg

在 PowerShell 中先执行：

```
winget search ffmpeg
```

找到 FFmpeg 软件包以后，通常可以执行：

```
winget install --id Gyan.FFmpeg -e
```

安装结束后，关闭 PowerShell，再重新打开，测试：

```
ffmpeg -version
```

出现版本信息就说明安装成功。WinGet 是 Windows 官方的软件包管理工具；FFmpeg 则负责读取、转换和重新编码音视频。

### 第二步：把字幕复制到视频目录

最终目录保持简单：

```
D:\faster_whisper\
├── test.mp4
├── test.srt
└── transcribe.py
```

为了避免 Windows 路径转义问题，建议先这样放置。

### 第三步：执行烧录命令

进入该目录：

```
cd D:\faster_whisper
```

执行：

```
ffmpeg -i test.mp4 -vf "subtitles=test.srt" -c:v libx264 -crf 23 -preset medium -c:a copy test_with_subtitles.mp4
```

完成以后会生成：

```
test_with_subtitles.mp4
```

这就是带永久字幕的新视频。FFmpeg 的 `subtitles` 滤镜会使用字幕时间轴将文字绘制到视频画面上。

## 字幕中文乱码时

先确认你的 `test.srt` 是 UTF-8 编码。

用记事本打开 `test.srt`：

```
文件 → 另存为
```

编码选择：

```
UTF-8
```

再重新运行 FFmpeg。

也可以指定中文字体：

```
ffmpeg -i test.mp4 -vf "subtitles=test.srt:force_style='FontName=Microsoft YaHei,FontSize=18,Outline=1,Shadow=0,MarginV=25'" -c:v libx264 -crf 23 -preset medium -c:a copy test_with_subtitles.mp4
```

这里使用的是微软雅黑。

## 你的电脑建议使用的参数

你的电脑没有 NVIDIA 显卡，所以不要使用：

```
h264_nvenc
```

使用 CPU 编码即可：

```
-c:v libx264
```

速度与画质可以这样调整：

```
-preset fast
```

速度更快，但文件可能略大。

```
-preset medium
```

速度和文件大小比较均衡。

```
-crf 18
```

画质较高，文件较大。

```
-crf 23
```

默认推荐，画质与大小均衡。

```
-crf 28
```

文件更小，但画质会下降。

你现在最适合直接使用：

```
ffmpeg -i test.mp4 -vf "subtitles=test.srt:force_style='FontName=Microsoft YaHei,FontSize=18,Outline=1,MarginV=25'" -c:v libx264 -crf 23 -preset fast -c:a copy test_with_subtitles.mp4
```

建议先做外挂字幕测试，因为几乎立即完成；确认字幕内容和时间轴正确以后，再烧录到视频中。

接下来建议先检查三件事：

1. 字幕内容是否准确
   - 专业名词有没有识别错
   - 中文标点是否自然
   - 有没有漏句、重复句
2. 时间轴是否同步
   - 字幕是否提前或延后
   - 长句是否显示时间太短
   - 视频中静音部分是否生成了错误文字
3. 字幕编码是否正常
   - 中文没有乱码
   - SRT 文件使用 UTF-8 编码

如果只是自己看视频，**外挂字幕已经足够了**，而且比烧录字幕更好：

- 不重新编码，不损失画质
- 处理速度快
- 字幕可以随时修改或关闭
- 视频文件不会变大
