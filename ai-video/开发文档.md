## 1. 项目概述

### 1.1 项目背景

AI Video Learning Assistant 是一个围绕"视频学习"场景构建的 AI 工具链。用户上传无字幕的教学/课程视频后，系统自动完成语音识别、字幕生成、字幕烧录、AI 总结、学习笔记生成等一系列操作，最终输出结构化的学习资料。

### 1.2 项目目标

| 目标 | 说明 |
|------|------|
| **MVP** | 视频 → 语音识别 → TXT + SRT → 学习笔记 |
| **V2** | 自动烧录字幕到视频 |
| **V3** | Web 界面 + 多视频批量处理 |
| **V4** | RAG 知识库 + 对话问答 |

### 1.3 项目定位

- **用途**：个人学习工具 / 课程视频处理 / AI 学习笔记
- **用户**：学生、自学者、在线课程学习者
- **运行环境**：Windows（无 NVIDIA 显卡，CPU 推理）

## 4. 模块说明

### 4.1 transcribe.py — 语音识别模块

**文件位置**：`测试/transcribe.py`

**功能**：将视频文件的语音转换为带时间轴的文字。

**核心流程**：

```
加载 faster-whisper 模型 (small, int8, cpu)
  → 读取视频文件
  → 调用 model.transcribe()
  → 遍历 segments
    → 写入 TXT（纯文本，不含时间）
    → 写入 SRT（含时间轴的标准字幕格式）
```

**关键参数**：

| 参数 | 当前值 | 说明 |
|------|--------|------|
| model_size | small | 模型大小（tiny/base/small/medium/large） |
| device | cpu | 推理设备 |
| compute_type | int8 | CPU 量化精度 |
| language | zh | 指定语言提高准确率 |
| beam_size | 5 | 束搜索宽度，越大越准越慢 |
| vad_filter | True | 语音活动检测过滤静音 |

**用法**：
```bash
python transcribe.py "D:\videos\课程视频.mp4"
```

### 4.2 generate_notes.py — 学习笔记生成模块

**文件位置**：`generate_notes.py`

**功能**：将 ASR 输出的纯文本通过大模型整理为结构化学习笔记。

**核心流程**：

```
读取 TXT 文件
  → 检查 Ollama 服务状态
  → 文本分段（长文本自动拆分）
    → 若单段 ≤ 4000 字符：直接调用模型生成
    → 若多段：逐段摘要 → 合并摘要 → 最终生成
  → 输出 Markdown 学习笔记
```

**输出结构**（严格的 Markdown 格式）：

| 章节 | 内容 |
|------|------|
| 📋 课程概述 | 2-3 句概括核心内容和学习目标 |
| 🎯 核心知识点 | 3-6 个重要知识点 |
| 📝 详细笔记 | 按逻辑分小节展开整理 |
| 🔑 重点术语 | 专业术语表格（如有） |
| ❓ 复习思考题 | 3-5 道简答题 |

**用法**：
```bash
python generate_notes.py test.txt
python generate_notes.py test.txt --model qwen2.5:3b --chunk-size 4000
```

**依赖**：Ollama 服务必须在本地运行（`ollama serve`）。

### 4.3 生成笔记.bat — Windows 快捷入口

**功能**：封装了 Python 环境检查和参数交互的批处理脚本，双击即可使用。

## 8. 已知限制与风险

| 限制 | 说明 | 缓解方案 |
|------|------|----------|
| CPU 推理慢 | 无 GPU，small 模型转录 10 分钟视频约需 5-10 分钟 | 使用 tiny 模型加速（精度略降） |
| 3B 模型能力有限 | 复杂课程理解可能不够深入 | 考虑升级到 7B（需 8GB+ 内存） |
| 无 Web 界面 | 当前只有 CLI | 后续用 FastAPI + Vue3 构建 |
| 单视频处理 | 不支持批量 | 后续加入队列/批处理 |
| 视频占用空间 | 大视频文件消耗存储 | 增加清理策略提示 |
