# AI Video Learning Assistant — 开发流程指南

> 版本：v1.0 | 生效日期：2026-07-11

本文档描述从"有一个想法"到"发布新版本"的完整开发流程。这是一个单人项目，所以流程轻量但结构清晰。

---

## 1. 开发工作流总览

```
[想法/需求] → [分析设计] → [编码实现] → [测试验证] → [发布归档]
     ↑                                                │
     └──────────────── 反馈循环 ──────────────────────┘
```

---

## 2. 阶段一：需求与分析

### 2.1 什么时候开始

- 有一个明确的使用场景（如"我想支持批量处理多个视频"）
- 发现了一个实际问题（如"SRT 编码导致字幕乱码"）
- 有了一个改进灵感（如"用 Web 界面替代命令行"）

### 2.2 做什么

1. **写需求说明**（几句话即可，不必长篇大论）
   - 我要解决什么问题？
   - 用户如何使用这个功能？
   - 预期输入/输出是什么？

2. **评估可行性**
   - 当前技术栈能否实现？
   - 有没有依赖/硬件限制？
   - 需要学习什么新东西？

3. **决定是否做**

### 2.3 产出物

一个清晰的描述，记录在项目文档或 Issue 中。例如：

```
## 需求：支持 VTT 字幕输出

场景：VTT 是 Web 视频播放器的标准字幕格式，后续 Web UI 需要用到。
输入：transcribe.py 现有的 segments 数据
输出：outputs/test.vtt 文件
方案：在 transcribe.py 中添加 write_vtt() 函数，格式参考 W3C 标准
```

---

## 3. 阶段二：设计与规划

### 3.1 确认目标

回答三个问题：

| 问题 | 示例回答 |
|------|----------|
| 这次改什么？ | 在 transcribe.py 增加 VTT 输出 |
| 不改什么？ | 不修改其它模块，不改变现有 TXT/SRT 行为 |
| 怎么验证？ | 运行后检查 outputs/ 下有 .vtt 文件，内容格式正确 |

### 3.2 设计接口

如果是新功能，想清楚它的输入输出：

```python
# 新增函数设计
def write_vtt(segments: list[Segment], output_path: Path) -> None:
    """将 segments 写为 VTT 格式字幕文件。

    Args:
        segments: faster-whisper 的 segment 列表
        output_path: 输出文件路径
    """
```

### 3.3 检查依赖

- 新增第三方库？→ 添加到 `requirements.txt`
- 需要新工具（如 FFmpeg）？→ 记录到文档
- 会影响现有功能？→ 确认兼容性

### 3.4 产出物

- 设计说明（几句话或一段伪代码）
- 如有必要，更新 `开发文档.md` 中的模块说明

## 4. 阶段三：编码实现

### 4.1 标准操作流程

```
1. git checkout dev          # 切换到开发分支
2. git pull                  # 同步最新代码
3. git checkout -b feat/xxx  # 创建功能分支
```

### 4.2 编码

按以下顺序进行，推荐 TDD（测试驱动开发）：

```
1️⃣ 写测试（定义"什么是对的"）
   ↓
2️⃣ 写实现（让测试通过）
   ↓
3️⃣ 重构（让代码变好）
   ↓
4️⃣ 提交（记录变更）
```

**TDD 示例**：

```python
# 第 1 步：先写测试
def test_format_timestamp():
    from transcribe import format_timestamp
    assert format_timestamp(0.0) == "00:00:00,000"
    assert format_timestamp(3661.5) == "01:01:01,500"

# 第 2 步：实现
def format_timestamp(seconds: float) -> str:
    ms = int(round(seconds * 1000))
    h, ms = divmod(ms, 3_600_000)
    m, ms = divmod(ms, 60_000)
    s, ms = divmod(ms, 1_000)
    return f"{h:02d}:{m:02d}:{s:02d},{ms:03d}"

# 第 3 步：运行测试 → 通过

# 第 4 步：提交
# git commit -m "feat(asr): 添加 format_timestamp 工具函数"
```

### 4.3 提交规则

- **频繁提交**：完成一个逻辑步骤就提交
- **提交前**：检查是否有多余的 print()、调试代码、硬编码路径
- **提交信息**：遵循 Conventional Commits 格式（详见 `开发规范.md`）

```
git add transcribe.py
git commit -m "feat(asr): 添加 VTT 字幕格式输出"
```

### 4.4 代码审查（自我审查清单）

编码完成后，逐一检查：

```
□ 类型注解完整（所有函数参数和返回值）
□ 函数有 docstring（说明功能、参数、返回值）
□ 没有硬编码的路径或魔术数字
□ 异常有正确处理（不会静默失败）
□ 代码风格符合 PEP 8
□ 新增依赖已添加到 requirements.txt
□ 没有遗留的调试 print()
```

## 5. 阶段四：测试验证

### 5.1 单元测试

针对工具函数和独立逻辑：

```bash
# 运行所有测试
python -m pytest tests/ -v

# 运行单个测试文件
python -m pytest tests/test_utils.py -v
```

### 5.2 集成测试

验证模块间的配合：

```bash
# 1. 转录音视频
python transcribe.py 测试/test.mp4

# 2. 检查输出
ls outputs/  # 应有 test.txt, test.srt

# 3. 生成笔记
python generate_notes.py outputs/test.txt

# 4. 检查笔记
ls outputs/  # 应有 test_学习笔记.md
```

### 5.3 人工验证

对 AI 相关的输出（ASR 准确率、笔记质量）需要人工检查：

```
□ 字幕时间轴是否同步？
□ 识别的文字有没有明显错误？
□ 笔记的课程概述是否准确？
□ 核心知识点是否覆盖了课程重点？
□ 术语解释是否正确？
```

### 5.4 边缘情况检查

```
□ 空视频 / 无语音的视频
□ 超长文本（ > 8000 字符）
□ 混合中英文的视频
□ 有背景噪音的视频
□ 不同采样率/格式的音频
```

## 6. 阶段五：合并与发布

### 6.1 合并到开发分支

```bash
# 确保功能分支已提交所有变更
git checkout dev
git merge feat/xxx
```

### 6.2 版本发布

当 dev 分支足够稳定时发布到 main：

```bash
git checkout main
git merge dev
git tag v1.1.0      # 打版本标签
```

版本号规则（语义化版本）：

| 版本 | 说明 | 示例 |
|------|------|------|
| v1.0.0 | 初始稳定版 | 首次发布 |
| v1.1.0 | 新增功能（向后兼容） | 添加 Web UI |
| v1.1.1 | 修复 bug | 修复字幕时间偏移 |
| v2.0.0 | 不兼容的重大变更 | 架构重构 |

### 6.3 发布检查清单

```
□ 所有代码已合并到 main
□ 核心功能经过完整测试
□ 文档已更新（开发文档/README）
□ 版本标签已打
□ 已知问题已记录
```

## 7. 日常开发节奏建议

### 7.1 一个典型的开发日

```
1. 看 issue/需求列表 → 选一个任务
2. 切分支 → 编码 → 测试 → 提交
3. 合并到 dev
4. 记录今天做了什么（更新笔记/文档）
```

### 7.2 遇到卡住怎么办

```
1. 搜索文档（查看开发文档.md）
2. 搜索类似问题（Google/Stack Overflow）
3. 简化问题：去掉复杂部分，先跑通最简路径
4. 记录 TODO：先绕过，回头再解决
```

### 7.3 保持项目健康

```
每日：
  □ 以一个提交结束（不要留下未提交的代码）

每次新功能：
  □ 更新文档（代码改了，文档也要改）
  □ 更新 dependencies（requirements.txt）

定期：
  □ 清理 outputs/ 中的旧文件
  □ 检查是否有过多未合并的分支
```

## 8. 快速参考：常用命令

### 8.1 开发命令

```bash
# 语音识别
python 测试/transcribe.py "视频文件.mp4"

# 生成笔记
python generate_notes.py outputs/test.txt

# 字幕烧录
ffmpeg -i test.mp4 -vf "subtitles=test.srt" \
  -c:v libx264 -crf 23 -preset fast \
  -c:a copy test_with_subtitles.mp4
```

### 8.2 Git 命令

```bash
git checkout -b feat/xxx      # 创建功能分支
git add <file>                # 暂存文件
git commit -m "feat: xxx"     # 提交
git checkout dev && git merge feat/xxx  # 合并
git tag v1.0.0                # 打标签
```

### 8.3 测试命令

```bash
python -m pytest tests/ -v              # 全部测试
python -m pytest tests/test_utils.py    # 单文件测试
python -m pytest -k "timestamp" -v      # 按关键词筛选
```
