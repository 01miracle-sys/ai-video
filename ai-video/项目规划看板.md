## 📊 技术债务跟踪

| 债务项 | 影响 | 计划解决 |
|--------|------|----------|
| 硬编码配置 | 修改参数需要改代码 | V2 阶段抽取到 YAML |
| 无单元测试 | 重构风险高 | MVP 收尾阶段增加 |
| 单文件脚本 | 代码复用困难 | MVP 收尾阶段重构 |
| print() 日志 | 难以控制输出级别 | MVP 收尾阶段改为 logging |
| 缺少 .gitignore | 容易误提交临时文件 | 立即解决 |

## 📝 如何使用这个看板

1. **选择一个任务** → 比如"把配置抽取到 YAML"
2. **切到 dev 分支** → `git checkout -b feat/config-yaml`
3. **编码实现** → 按照 `开发流程.md` 的步骤
4. **测试验证** → 确保原有功能不受影响
5. **合并提交** → 合并到 dev，并在这里勾选 ☑️
6. **记录进展** → 在这里更新完成状态
